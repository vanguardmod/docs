[[cvars]]
= Cvar Reference

This chapter is the alphabetical, comprehensive reference for every cvar
relevant to a {vg-name} server administrator. {vg-name}'s own cvars
(`vg_*` and `vanguard_*`) are documented in full. The {etl-name}-inherited
cvars that a {vg-name} admin must understand on day one are also
documented here, in <<cvars-inherited>>, with their {etl-name} defaults
and any {vg-name}-specific cup-mode behaviour called out.

[NOTE]
====
**Convention** — every cvar lives in its own file under `cvar/` in the docs
repository, modelled on {mod-jaymod}'s `doc/cvar/` layout. New cvars must be
`include::`d below in alphabetical order.

The `Origin` row in each cvar's header table identifies where the cvar
comes from:

* **Engine ({et-name} / {etl-name})** — server-engine cvars: `sv_*`,
  `com_*`, `net_*`, `fs_*`.
* **Game ({etl-name})** — game-module cvars: `g_*`, `match_*`,
  `vote_allow_*`.
* **Game ({etl-name}, overridden by {vg-name})** — game cvars whose
  default the {vg-name} mod changes from upstream.
* **{vg-name}** — added by {vg-name}: `vg_*`, `vanguard_*`.
====

== Inherited from ET / ETLegacy

apply to {vg-name} unchanged unless explicitly noted. They are
documented here because a {vg-name} server administrator needs them on
day one — every `server.cfg` that boots a usable {vg-name} server
touches a substantial subset of this list.

The grouping below is by purpose, not by alphabet, so that a newcomer
can read top-to-bottom and end with a working mental model of a
`server.cfg`. The annotated profiles in <<config-walkthrough>> use the
