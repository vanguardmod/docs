[[cvar.rconPassword]]
=== rconPassword

[cols="1h,4",frame=all,grid=rows]
|===
| Default       | `""` (empty)
| Type          | string
| Flags         | `CVAR_TEMP`
| Origin        | Engine ({et-name} / {etl-name})
| In cup mode   | tunable
| Scope         | server
|===

Password that authorises remote-server-command (`rcon`) requests. With
`rconPassword` empty (the engine default) **all `rcon` traffic is
refused**, which is also the engine's safe default — every `server.cfg`
must set this if you want to administer the server from a connected
client.

The `_TEMP` flag means the value is not persisted to `etconfig.cfg`;
it must be set in `server.cfg` on every boot.

[CAUTION]
====
The rcon password is the keys-to-the-kingdom: anyone who has it can
issue any server command, including ones that change configuration,
ban players, or shut the server down.

* Pick a long, random secret. Never reuse a password from elsewhere.
* Do not commit the live value to any Git repository.
* Rotate after every staff change.
* Prefer connecting from a trusted host: rcon traffic is in plaintext
  over UDP, so anyone who can sniff the wire and the password can
  impersonate you.
====

==== Examples

.Set in `server.cfg`
[source]
----
set rconPassword "CHANGEME-rcon"
----

.Use it from the in-game client console
[source,console]
----
] /rconpassword CHANGEME-rcon
] /rcon status
----
