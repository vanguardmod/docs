[[cvar.sv_privatePassword]]
=== sv_privatePassword

[cols="1h,4",frame=all,grid=rows]
|===
| Default       | `""` (empty)
| Type          | string
| Flags         | `CVAR_TEMP`
| Origin        | Engine ({et-name} / {etl-name})
| In cup mode   | tunable
| Scope         | server
|===

Password that grants a connecting client one of the
<<cvar.sv_privateClients,reserved slots>>. Distributed out-of-band to
admins, refs, or friends. Empty by default; if `sv_privateClients` is
non-zero you must set this, otherwise the reserved slots can never be
filled.

The `_TEMP` flag means the value is **not** written to `etconfig.cfg` on
shutdown — it must be set explicitly in `server.cfg` (or via `rcon`)
each boot.

[CAUTION]
====
Treat this like any other shared secret. Do not commit a real value to
a public Git repository, and rotate it whenever someone with access
leaves your team.
====

==== Examples

.Set in `server.cfg`
[source]
----
set sv_privateClients  4
set sv_privatePassword "CHANGEME-private"
----
